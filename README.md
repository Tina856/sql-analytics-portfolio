## SQL Session 01 Summary

 ### Ի՞նչ է տվյալների բազան (Database)

Տվյալների բազան տվյալների պահպանման և կազմակերպման համակարգ է, որտեղ տվյալները դասավորվում են աղյուսակներով՝ տողերի (rows) և սյուների (columns) ձևով։
SQL տվյալների բազաները ապահովում են արագ որոնում, մեծածավալ տվյալների արդյունավետ կառավարում և միաժամանակյա աշխատանք մի քանի օգտատիրոջ կողմից։

 ### Spreadsheets vs Databases

Քննարկեցինք Excel-ի նման աղյուսակային ֆայլերի (spreadsheets) և տվյալների բազաների տարբերությունները․
- Spreadsheets նախատեսված է հիմնականում մեկ օգտատիրոջ համար
- Databases հնարավորություն է տալիս աշխատել մեծ տվյալների հետ և միաժամանակ սպասարկել մի քանի օգտատիրոջ

Databases-ում տվյալները ավելի անվտանգ և կառուցված են։

 ### SQL-ի սկզբնական պատկերացում (Intro to SQL)

SQL (Structured Query Language) լեզուն օգտագործվում է տվյալների բազաներում հարցումներ կատարելու և տվյալները մշակելու համար։
SQL-ի միջոցով կարող ենք․
- ընտրել տվյալներ (SELECT)
- ֆիլտրել եւ հաշվարկել (WHERE, COUNT, SUM, AVG և այլն)
- դասակարգել և սահմանափակել (ORDER BY, LIMIT)
- կառուցել բարդ որոնողական պայմաններ։

### Կառուցված և Չկառուցված տվյալներ (Structured vs Unstructured Data)

- Կառուցված տվյալները ունեն հստակ կառուցվածք՝ տողեր և սյուներ (օրինակ՝ SQL աղյուսակներ)
- Չկառուցված տվյալները չունեն նախորոշված կառուցվածք (օրինակ՝ նկարներ, վիդեո, ազատ տեքստ)
Այդ երկու տեսակների տարբերությունը կարևոր է հասկանալու համար, թե որ տվյալը ինչպես պետք է մշակվի։


## SQL Session 02 Summary

### Տվյալների ընտրություն և ֆիլտրում (Filtering Data)

Այս սեսիայում խորացանք տվյալների ընտրության և ֆիլտրելու մեխանիզմների մեջ։

WHERE — տվյալների ֆիլտրում պայմաններով

Համեմատական օպերատորներ՝ =, !=, >, <, >=, <=

Տրամաբանական օպերատորներ՝ AND, OR, NOT

IN, BETWEEN — բազմակի արժեքներով և միջակայքով ֆիլտրում

LIKE — տեքստային որոնում pattern-ներով (%, _)


### Արժեքների տեսակներ (Data Types)

Թվային (INTEGER, NUMERIC)

Տեքստային (TEXT, VARCHAR)

Ամսաթվային (DATE, TIMESTAMP)

Քննարկեցինք, թե ինչպես են տիպերը ազդում հաշվարկների և համեմատությունների վրա։


### Դասակարգում և սահմանափակում (Sorting & Limiting)

ORDER BY — արդյունքների դասակարգում (ASC, DESC)

LIMIT — արդյունքների քանակի սահմանափակում

OFFSET — բաց թողնել առաջին N տողերը


## SQL Session 03 Summary

### Ագրեգացիա և խմբավորում (Aggregation & Grouping)

Սովորեցինք հաշվարկել ընդհանուր արժեքներ և խմբավորել տվյալները։

Ագրեգատ ֆունկցիաներ՝

COUNT()

SUM()

AVG()

MIN(), MAX()

GROUP BY — տվյալների խմբավորում ըստ սյունակի

HAVING — խմբերի ֆիլտրում (ագրեգացիայից հետո)

Տարբերությունը WHERE և HAVING միջև՝

WHERE → ֆիլտրում է տողերը մինչև խմբավորումը

HAVING → ֆիլտրում է արդեն խմբավորված արդյունքները


### Աշխատանք NULL արժեքների հետ

IS NULL, IS NOT NULL

Ինչպես են NULL-երը ազդում հաշվարկների վրա

Ինչու COUNT(column) ≠ COUNT(*) երբ կան NULL-եր
